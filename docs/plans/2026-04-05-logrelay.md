# LogRelay 分阶段实现计划

## 概述

基于设计文档，分 6 个阶段实现 LogRelay。每阶段独立可验证，完成后即可用于日常工作。

**核心原则**：先跑通最小闭环（Phase 1-3），再逐步扩展适配器和可视化。

---

## Phase 1: 核心模块 + 模板

**目标**：搭建 `src/core/` 和 `templates/`，为所有上层模块提供共享基础。

**产出物**：
- `src/core/config.py` — 全局配置
- `src/core/formatter.py` — 日志格式化
- `src/core/status_manager.py` — STATUS.md 读写
- `src/core/utils.py` — 工具函数
- `templates/log_template.md` — 日志文件模板
- `templates/status_template.md` — STATUS.md 模板

### Task 1.1: utils.py — 通用工具函数

**文件**：`src/core/utils.py`

实现内容：
- `find_vault_root(start_path: Path) -> Path`：从给定路径向上遍历，找到 vault 根目录（标志：包含 `my_project/` 或 `.obsidian/`）
- `find_project_root(start_path: Path, vault_root: Path) -> tuple[Path, str]`：定位项目目录和项目名（vault 直接子目录）
- `ensure_logs_dir(project_path: Path, tool_name: str) -> Path`：确保 `项目/logs/工具名/` 目录存在
- `generate_filename(summary_keywords: str = "") -> str`：生成 `{YYYY-MM-DD}_{HHMMSS}-{摘要关键词}.md` 格式文件名
- `get_timestamp() -> str`：ISO 8601 格式时间戳
- `normalize_path(path: Path) -> str`：路径统一为正斜杠字符串

**验证**：单元测试覆盖项目定位的各种场景（vault根、项目根、子目录、深层嵌套）。

### Task 1.2: config.py — 全局配置

**文件**：`src/core/config.py`

实现内容：
- `SUPPORTED_TOOLS = ["claude-code", "trae", "cursor", "qoder"]`
- `LOGS_DIR_NAME = "logs"`
- `STATUS_FILENAME = "STATUS.md"`
- `get_tool_config(tool_name: str) -> dict`：返回工具特定配置（hooks路径、commands路径等）
- `get_vault_root() -> Path`：基于 `__file__` 定位 vault 根（`src/core/config.py` → 向上3级到 vault）

### Task 1.3: formatter.py — 日志格式化

**文件**：`src/core/formatter.py`

实现内容：
- `create_log_content(frontmatter: dict, summary: str, tasks: list, decisions: list, artifacts: list, context: dict, conversation: str = "") -> str`：组装完整日志 Markdown
- `parse_frontmatter(content: str) -> dict`：解析日志 frontmatter
- `format_frontmatter(data: dict) -> str`：将 dict 序列化为 YAML frontmatter 字符串
- `update_ended_status(content: str, ended_time: str, status: str) -> str`：更新 ended 和 status 字段

依赖：utils.py

### Task 1.4: status_manager.py — STATUS.md 管理

**文件**：`src/core/status_manager.py`

实现内容：
- `read_status(project_path: Path) -> dict | None`：读取并解析 STATUS.md
- `write_status(project_path: Path, data: dict) -> None`：写入/更新 STATUS.md
- `create_initial_status(project_path: Path, project_name: str) -> None`：创建初始 STATUS.md
- `update_status_on_session_end(project_path, session_info: dict) -> None`：会话结束时更新
- `get_pending_tasks(status: dict) -> list`：提取未完成任务
- `get_recent_sessions(status: dict, n: int = 5) -> list`：获取最近 N 条会话记录
- `add_session_history(status: dict, session: dict) -> dict`：添加会话历史记录
- `update_artifacts_index(status: dict, artifacts: list) -> dict`：更新产出物索引

依赖：formatter.py, utils.py

### Task 1.5: 模板文件

**文件**：
- `templates/log_template.md`
- `templates/status_template.md`

内容：与设计文档中的格式示例一致，用 `{placeholder}` 标记可替换字段。

## Phase 2: Hooks + Commands

**目标**：实现会话生命周期管理，跑通 Claude Code 的自动日志记录。

**产出物**：
- `src/hooks/session_start.py`
- `src/hooks/session_end.py`
- `src/hooks/post_tool_use.py`
- `src/commands/start_log.py`
- `src/commands/end_log.py`

### Task 2.1: session_start.py — 会话开始 Hook

**文件**：`src/hooks/session_start.py`

实现内容：
- CLI 入口，`--tool` 参数指定工具名
- 调用 `utils.find_vault_root()` 和 `utils.find_project_root()` 定位项目
- 读取 STATUS.md（如存在），输出未完成任务到 stdout（供 hook 注入上下文）
- 创建新日志文件（frontmatter 中 status=active），写入初始内容
- 输出上下文信息格式：
  ```
  [LogRelay] 项目: {项目名}
  [LogRelay] 检测到上次会话（{工具}, {日期}）有未完成任务：
  [LogRelay]   - {任务1}
  [LogRelay]   - {任务2}
  [LogRelay] 相关日志: {路径}
  ```
- 如果无 STATUS.md，输出 `[LogRelay] 新项目，无历史会话。`

依赖：Phase 1 全部模块

**验证**：在 vault 子目录下手动运行，确认输出正确。

### Task 2.2: session_end.py — 会话结束 Hook

**文件**：`src/hooks/session_end.py`

实现内容：
- CLI 入口，`--tool` 参数 + `--session-id` + `--summary`（可选，从 stdin 读取摘要）
- 摘要来源：
  1. `--summary` 参数直接传入
  2. stdin 管道传入（AI 输出管道）
  3. 无摘要时自动生成基础摘要（时间+工具+项目名）
- 更新日志文件：填入 ended 时间、status、摘要、任务清单等
- 更新 STATUS.md：记录未完成任务、会话历史、产出物索引

依赖：Phase 1 全部模块

### Task 2.3: post_tool_use.py — 工具调用后 Hook（可选）

**文件**：`src/hooks/post_tool_use.py`

实现内容：
- CLI 入口，`--tool` 参数
- 追加对话片段到当前活跃日志文件
- 轻量实现：仅在日志末尾追加时间戳+操作摘要
- 设计为可选功能，不影响核心流程

依赖：Phase 1

### Task 2.4: start_log.py / end_log.py — CLI 命令

**文件**：`src/commands/start_log.py`, `src/commands/end_log.py`

实现内容：
- 与 hooks 版本逻辑相同，但面向手动调用（Cursor/Qoder）
- `start_log.py`：执行 session_start 的全部逻辑，额外输出提示信息引导用户
- `end_log.py`：执行 session_end 的全部逻辑，支持交互式输入摘要

依赖：Phase 1 + Phase 2 hooks

## Phase 3: 适配器 + 安装脚本

**目标**：实现一键部署，让 LogRelay 在 Claude Code 上可用。

**产出物**：
- `src/adapters/claude_code.py`
- `src/adapters/trae.py`
- `src/adapters/cursor.py`
- `src/adapters/qoder.py`
- `install.py`

### Task 3.1: claude_code.py — Claude Code 适配器

**文件**：`src/adapters/claude_code.py`

实现内容：
- `generate_hooks_config(script_root: Path) -> dict`：生成 `.claude/settings.json` 中的 hooks 配置
- `install(vault_root: Path) -> None`：写入 hooks 配置到 `.claude/settings.json`（合并而非覆盖）
- `uninstall(vault_root: Path) -> None`：移除 hooks 配置
- 平台适配：`python` (Windows) vs `python3` (macOS)

### Task 3.2: trae.py — Trae 适配器

**文件**：`src/adapters/trae.py`

实现内容：
- `generate_hooks_config(script_root: Path) -> dict`
- `install(vault_root: Path) -> None`：写入 `.trae/skills/logrelay/hooks.json`
- `uninstall(vault_root: Path) -> None`

### Task 3.3: cursor.py — Cursor 适配器

**文件**：`src/adapters/cursor.py`

实现内容：
- `install(vault_root: Path) -> None`：
  - 写入 `.cursor/commands/start-log.md`
  - 写入 `.cursor/commands/end-log.md`
  - 写入 `.cursor/skills/logrelay/logrelay-rule.md`
- `uninstall(vault_root: Path) -> None`
- commands 内容包含：执行 Python 脚本的指令 + 引导 AI 生成摘要的提示

### Task 3.4: qoder.py — Qoder 适配器

**文件**：`src/adapters/qoder.py`

实现内容：
- 与 Cursor 类似结构
- 写入 `.qoder/commands/` 和 `.qoder/agents/`

### Task 3.5: install.py — 一键安装脚本

**文件**：`install.py`（项目根目录）

实现内容：
- CLI 入口：`--tools`（逗号分隔工具列表）、`--uninstall`（卸载模式）
- 执行流程：
  1. 检测平台（Windows/macOS）
  2. 检查 Python 版本 >= 3.8
  3. 检查各工具配置目录是否存在
  4. 逐个调用适配器的 install/uninstall
  5. 输出安装状态报告
- vault 根通过 `__file__` 相对定位
- 幂等：重复运行不产生重复配置

**里程碑验证**：
1. 运行 `python install.py --tools claude-code`
2. 确认 `.claude/settings.json` 中 hooks 配置正确
3. 在一个项目目录下启动 Claude Code
4. 确认 session_start 自动触发并输出上下文
5. 执行 `/end-log`，确认日志文件和 STATUS.md 正确生成

## Phase 4: Obsidian 插件

**目标**：实现三个核心视图，在 Obsidian 中可视化日志数据。

**产出物**：
- `src/obsidian-plugin/` 下全部 TypeScript 源码
- `obsidian-plugin/` 下构建产物

### Task 4.1: 插件骨架 + 设置页

**文件**：`src/obsidian-plugin/main.ts`, `src/obsidian-plugin/settings.ts`

实现内容：
- 插件入口：注册三个 View，加载设置
- 设置页：
  - 日志目录名（默认 `logs`）
  - 状态文件名（默认 `STATUS.md`）
  - 接力链路显示深度（默认 10）
  - 搜索索引间隔（默认 5 分钟）
  - 统计周期（默认 30 天）
- `manifest.json`：插件元数据

### Task 4.2: 日志解析服务

**文件**：`src/obsidian-plugin/services/log-parser.ts`

实现内容：
- `parseAllLogs(vault: Vault, settings: LogRelaySettings) -> LogEntry[]`：扫描所有项目下的日志
- `parseFrontmatter(content: string) -> FrontmatterData`：解析 YAML frontmatter
- `parseTasks(content: string) -> Task[]`：解析任务清单
- `parseConversation(content: string) -> ConversationEntry[]`：解析对话记录
- 使用 Obsidian 的 `MetadataCache` 加速 frontmatter 读取

### Task 4.3: 接力链路视图

**文件**：`src/obsidian-plugin/views/relay-view.ts`, `src/obsidian-plugin/components/relay-graph.ts`

实现内容：
- 侧边栏 View，选择项目后展示接力链路
- 从日志的 `上下文传递` 字段构建依赖图
- 节点颜色：完成=绿、进行中=蓝、暂停=黄、失败=红
- 点击节点跳转到对应 Markdown 文件
- 使用 CSS 绘制连接线，不引入第三方库

### Task 4.4: 跨项目搜索视图

**文件**：`src/obsidian-plugin/views/search-view.ts`, `src/obsidian-plugin/services/search-engine.ts`, `src/obsidian-plugin/components/search-result.ts`

实现内容：
- 搜索框 + 筛选器（工具、项目、状态、日期、标签）
- 全文搜索日志内容和 frontmatter
- 搜索索引定时刷新
- 结果列表点击跳转

### Task 4.5: 统计面板

**文件**：`src/obsidian-plugin/views/stats-view.ts`, `src/obsidian-plugin/components/charts.ts`

实现内容：
- 工具使用频率（条形图）
- 任务完成率统计
- 活跃度热力图（GitHub 风格，纯 CSS）
- 项目活跃度 TOP 5
- 纯 Obsidian API + CSS 实现，不引入第三方图表库

### Task 4.6: 构建与部署

**文件**：`esbuild.config.js`, `package.json`, `manifest.json`, `styles.css`

实现内容：
- esbuild 打包为单个 `main.js`
- `npm run build` 构建到 `obsidian-plugin/`
- `install.py` 集成：构建 + 复制到 `.obsidian/plugins/logrelay/`

覆盖：
- install.py 的配置生成和写入

## 执行优先级

```
Phase 1 (核心) → Phase 2 (Hooks) → Phase 3 (安装) → Phase 4 (插件) → Phase 5 (测试) → Phase 6 (文档)
                                      ↑
                                  里程碑：Claude Code 可用
```

**最快可用路径**：Phase 1 + 2 + 3 完成后，即可在 Claude Code 上日常使用日志接力功能。Phase 4（Obsidian 插件）为增强功能，可后续迭代。

## 技术约束提醒

- Python 3.8+，纯标准库，`pathlib.Path`
- Hook 脚本统一 `.py`，不用 `.sh`/`.bat`
- 路径统一正斜杠
- Obsidian 插件 TypeScript + esbuild，纯 CSS 图表
- 日志文件标准 Markdown，frontmatter 兼容 Obsidian 属性
